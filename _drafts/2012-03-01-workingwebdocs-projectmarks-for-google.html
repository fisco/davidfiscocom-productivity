--- layout: post title: 'WorkingWebDocs: ProjectMarks for Google Chrome' date: '2012-03-01T16:25:00.000-05:00' author: David Fisco tags: - Web applications - Google Chrome - bookmarks - cloud computing - project management modified_time: '2013-11-05T22:00:43.785-05:00' blogger_id: tag:blogger.com,1999:blog-842616179832437348.post-3921310391891408290 blogger_orig_url: http://productivity.davidfisco.com/2012/03/workingwebdocs-projectmarks-for-google.html --- ProjectMarks is my Google Chrome extension that allows you to organize the resources of your projects using Google Chrome bookmarks.  By integrating with a growing list of Web applications, ProjectMarks offers an interface providing efficient and effective project organization and management, giving you summary and preview information on a single project page.  It obviates the need of visiting all of your Web services for a project you want to review.


While ProjectMarks will list any bookmark, it has advanced support only for bookmarks that are "project-oriented".  For example, a bookmark for a collection of Google Docs is project-oriented and ProjectMarks provides special functionality for those who authorize their Google account with ProjectMarks.  But a bookmark to a news article is better placed in a Web service such as Diigo or Delicious, then the Diigo or Delicious service should be bookmarked into a project's folder.  Once you start using ProjectMarks this approach should become intuitive.

Currently, ProjectMarks supports the following web services
  • Blogger
  • Disqus
  • Google Calendar
  • Google Docs
  • Google Maps
  • Google Tasks
  • Stack Overflow
  • Twitter
  • Wikipedia
  • YouTube.
The goal is to always be expanding this list with project-oriented services, especially after the project graduates from beta status.

Update (November 5, 2013): After not looking at this code for a while, I've begun working on it again, due to a spike of interest in the project.  I should have a new, working alpha of the project in the coming weeks.

ProjectMarks is currently in alpha and is being offered only for early adopters who want to try it out.  (It is not yet available on the Chrome Store because it isn't finished.)  To use ProjectMarks, you will need to create a folder called "ProjectMarkss" in the "Bookmarks Bar" of your Google Chrome bookmarks.  You can add this manually, or once installed, ProjectMarks will offer you a dialog to create the folder.  Next, install the ProjectMarks extension (version 0.1.20) into your Google Chrome.  (This alpha does not automatically update; you will need to update manually.)  If you want the advanced services, activate them in the ProjectMarks user interface.  Any time you want to create a new project, create a new folder in the Projects folder or use ProjectMarks to create one for you.  The title you give your new folder will be the name of your new project.

Using ProjectMarks is all about capturing the correct bookmarks for your projects.  It's both a piece of software and a project management mindset.  To get an idea of the types of bookmarks ProjectMarks recognizes, read my article "Organizing Your Cloud by Project, Using Google Chrome Bookmarks".  A bookmark not covered in that article that ProjectMarks understands is the Google Calendar bookmark.  To capture a Google calendar for use in ProjectMarks:

  1. Open your Google calendar at https://calendar.google.com.  
  2. On the left-hand side, select the drop-down menu for the calendar you want to add to your project.  
  3. Select "Calendar settings".  
  4. Under "Calendar address", click on "HTML".  
  5. Bookmark the resulting page in one of your project folders.

Why organize projects around bookmarks instead of building a custom data format in the ProjectMarks software?  Because bookmarks are portable and can be accessed from anywhere.  For example, if you install the Xmarks Bookmarks Sync Chrome extension, you can synchronize your bookmarks to several browsers and the Xmarks website.  Even when you can't use the ProjectMarks extension, your bookmark data will still be accessible, so you'll always have access to your projects.
